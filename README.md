# Отчёт по лабораторной работе №4
**«Симуляция с пользовательскими коллекциями и псевдослучайной моделью»**

## Реализация

### 1. Иерархия классов предметной области
Созданы три класса:
- **LibraryItem** – базовый класс с полями: название, автор, год
- **Book** – класс книги, наследник LibraryItem, добавлены поля: жанр, ISBN
- **Journal** – класс журнала, наследник LibraryItem, добавлены поля: номер выпуска, издательство

### 2. Пользовательские коллекции
Реализованы две коллекции:

**BookCollection** – списковая коллекция через композицию:
- Поддерживает индексацию, срезы, итерацию
- Имеет методы добавления и удаления книг
- Реализует магические методы: __len__, __iter__, __getitem__, __contains__

**IndexDict** – словарная коллекция для индексации:
- Индексирует книги по ISBN, автору, году, жанру
- Автоматически обновляется при изменении основной коллекции
- Предоставляет методы поиска по разным критериям

### 3. Класс Library
Управляющий класс библиотеки содержит:
- Коллекцию книг (BookCollection)
- Индексы (IndexDict)
- Методы для добавления, удаления и поиска книг

Методы поиска используют IndexDict для эффективного доступа.

### 4. Псевдослучайная симуляция
Реализована функция run_simulation() с параметрами:
- steps – количество шагов симуляции
- seed – значение для воспроизводимости результатов

На каждом шаге выполняется одно из 7 случайных событий:
1. Добавление новой книги
2. Удаление случайной книги
3. Поиск по автору
4. Поиск по году
5. Поиск по ISBN
6. Попытка найти несуществующую книгу
7. Вывод статистики

Все события логируются в консоль.

### 5. Тестирование
Созданы модульные тесты для всех компонентов системы:
- Тесты классов Book, Journal, LibraryItem
- Тесты коллекций BookCollection и IndexDict
- Тесты класса Library
- Тесты симуляции

### 6. Интерфейс
Реализован основной файл main.py с консольным интерфейсом, позволяющим запускать симуляцию в двух режимах:
- Стандартный запуск
- Запуск с фиксированным seed
